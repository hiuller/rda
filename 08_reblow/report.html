Dividindo o conjunto de dados em grupos pelo percentual de sucata

Vamos percorrer o conjunto de dados com janelas de banda fixa (inicialmente 1%) e vamos procurar as bandas que absorvem grande n√∫mero de corridas transversalmente (entre todas as fam√≠lias). Sabemos pelo jittered plot que vamos encontrar tais fam√≠lias entre 14 e 16% de sucata. O comprimento da banda √© importante porque entre 14 e 16% temos 2% de diferen√ßa. Se a banda tiver esse mesmo tamanho estaremos mais propensos ao confundimento. Iremos usar, portanto, non overlapping bands.

O algor√≠tmo para varredura est√° resumido nas seguintes tarefas:

  • obter um valor m√©dio inicial
  • calcular o limite inferior como = valor inicial - (banda/2)
  • calcular o limite superior como = valor inicial + (banda/2)
  • contar quantos valores de cada fam√≠lia estao inclu√≠dos na banda
banda <- 1.0  
i <- seq(from=13, to=20, by=0.1)
result <- c()
for(x in i){
  intervalo <- c(x-(banda/2), x+(banda/2))
  temp <- as.numeric(tapply(as.numeric(sub$PERC_SUCATA > intervalo[1] & sub$PERC_SUCATA < intervalo[2]), sub$TP_CORR, sum))
  result <- rbind(result, c(x, temp))
